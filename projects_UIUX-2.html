＞作品PORTFILIO＞SMART TAXI APP

UI + UX creation from dcratch

APP ( iOS / Android )

SMART TAXI APP

快速多元叫車

公司為提升服務品質與叫車服務多元化，計畫以階段性先進行「代客叫車」，除了優化原ＡＰＰ使用者體驗介面設計外， 更希望為未來預計擴充之服務項目，一併納入系統規劃考量。

UX使用者體驗設計 Step 1

開發需求確認 & 解決方法

開發時程考量下，在初步需求訪談後，進行快速競品分析，確認開發功能定義後，著手敏捷式開發。

UX使用者體驗設計 Step 2

Wireframe / UI FLOW

開發時程考量，捨棄Flow Chart過程，直接繪製Wireframe畫面框架圖，並製作UI FLOW，操作演練可能情境與使用流程。
與需求單位、ＲＤ等實際操作講解說明，確認需求功能完整、操作流程順暢；針對此階段所得到的改善方向Feedback，進行優化的修改調整。

UI使用者介面設計

MOCK UP 介面設計

  • 為求統一，沿用舊系統色系與字元層級等定義
  • 改善不易解讀原ICON設計 / 創造ICON顏色層次感
  • 遵循平台Guideline（iOS Human Interface / Material Design)

UI設計亮點-1

完形心理學（接近性）

將上下車地點、時間輸入欄位，規劃在同一區域，改善原地圖頁輸入乘車資訊介面零散，並將未來服務 功能擴充納入規劃考量，讓使用者選擇服務項目後，填入乘車等資訊（使用動線由上而下），增加使用者體驗直覺流暢感受。

UI設計亮點-2

代親友叫車超方便

點取連結手機聯絡簿，輕輕鬆鬆選取乘客資訊，手動輸入或修改也可行。

UI設計亮點-3

乘車地點輸入好用又靈活

  • 乘客上下車地點輸入，可以手動輸入，輸入過程中，系統自動帶出關鍵字地點，協助快速選取。
  • 另有移動地圖選取方式（定位標不動），定位標上方自動帶入定位地點，按下定位確認完成選取。

UI設計亮點-4

同時為自己與朋友叫車 / 司機資訊讀取選擇

  • 原系統開發時未有「自己與代親友同時叫車」的使用情境需求，故必須改變原叫車成功後的狀態頁面呈現方式，讓使用者完成個人叫車後，輕鬆切換至代叫功能頁面。
  • 隱藏司機資訊設計，減少過多的資料負擔，將讀取選擇交由使用者控制。
